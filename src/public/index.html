How URL Shortening Works

1

User Authentication

User logs in via Google OAuth 2.0. Session is stored in Redis for persistence.

2

URL Submission

Long URL is validated and sent to the API. Rate limiting prevents abuse.

3

Alias Generation

Unique short alias is generated (or custom alias validated). Data stored in PostgreSQL.

4

Redirect & Track

When short URL is visited, user is redirected. Analytics captured: IP, device, location.
